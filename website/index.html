The Columnar Document Database
for the Edge

A hybrid relational/document/graph database built on Apache Parquet. MongoDB-style API with bidirectional relationships, time-travel queries, and first-class Cloudflare Workers support.

Get Started View on GitHub
$ npm install parquedb

Why ParqueDB?

Built for modern edge computing with the features you need

C

Columnar Storage

Parquet-native storage with 8-10x compression. Read only the columns you need with predicate pushdown for fast analytical queries.

W

Cloudflare Workers

First-class support for Workers runtime with R2 storage and Durable Objects. CQRS architecture optimized for edge computing.

G

Graph Relationships

Bidirectional relationships are indexed and traversable from either direction. No joins, no $lookup, no additional queries.

T

Time-Travel Queries

Built-in event sourcing with full audit history. Query your data as it existed at any point in time with point-in-time reconstruction.

M

MongoDB-Style API

Familiar query and update operators. Drop-in replacement for MongoDB with additional graph and time-travel capabilities.

J

Pure JavaScript

No WASM or native dependencies. Runs in Node.js, browsers, and Cloudflare Workers using hyparquet for Parquet read/write.

Simple, Powerful API

Get started with just a few lines of code

schema.ts
import { DB } from 'parquedb'

// Define your schema
const db = DB({
  User: {
    email: 'string!#',
    name: 'string',
    role: 'string'
  },
  Post: {
    title: 'string!',
    content: 'text',
    author: '-> User'
  }
})
app.ts
// Create entities
const user = await db.User.create({
  email: 'alice@example.com',
  name: 'Alice'
})

// Query with MongoDB-style filters
const posts = await db.Post.find({
  author: user.$id
})

// Update with operators
await db.Post.update(posts[0].$id, {
  $set: { title: 'Updated' }
})
